if you play fortnite, you've probably seen leakers on twitter posting upcoming skins and shop rotations before they officially drop. ever wondered how that works? i built a system that tracks all of it automatically

what it actually does

the project is pretty simple at its core. a github actions workflow runs every 30 minutes, fetches data from fortnite's public API endpoints, and commits the changes to a git repo. that's it. no reverse engineering, no game file extraction. just publicly available data, tracked over time

the magic is in the diff. because everything is version-controlled, you can look at any commit and see exactly what changed — a new skin got added, the shop rotated, a playlist went live. git becomes a time machine for fortnite's data

what gets tracked

right now it monitors 7 different data sources:

  • battle royale cosmetics (skins, emotes, pickaxes, everything)
  • item shop rotations
  • in-game news across different modes
  • available playlists and gamemodes
  • AES encryption keys
  • player banners
  • build version metadata

all of this comes from fortnite-api.com, which aggregates fortnite's public API data into endpoints

the setup

the whole thing is written in typescript. the fetch logic is straightforward, hit the API, compare with what's already in the data/ directory, and if something changed, write the new JSON and let git pick up the diff:

// simplified version of what happens each run
const endpoints = [
  { name: "cosmetics", url: "/v2/cosmetics/br" },
  { name: "shop", url: "/v2/shop/br" },
  { name: "news", url: "/v2/news" },
  // ... more endpoints
];

for (const endpoint of endpoints) {
  const response = await fetch(BASE_URL + endpoint.url);
  const data = await response.json();
  writeFileSync(`data/${endpoint.name}.json`, JSON.stringify(data, null, 2));
}

the github action handles the rest (if any files changed, it commits and pushes automatically)

why git is perfect for this

i could've stored this in a database. but git gives you something a database doesn't: a complete, browsable history of every single change with timestamps. want to know when a specific skin was first added to the files? git log --all -p -- data/cosmetics.json and search for it

the commit messages are automated too, so you can scan through the history and immediately see what changed without opening any files

running it yourself

if you want to try it locally:

npm install
npm run fetch

that'll pull the latest data from all endpoints into your local data/ directory. fork the repo and enable github actions if you want your own automated tracker
